# Права и обязанности

| Раздел | Формулировка | Грейд |
|---|---|---|
| Лидерство | Формирует слаженную и результативную команду. | M1 |
| Лидерство | Разделяет цели компании, вдохновляя и мотивируя других. | M1 |
| Лидерство | Мотивирует сотрудников на постоянное улучшение себя и процессов. | M1 |
| Лидерство | Создаёт атмосферу доверия, чтобы все могли открыто высказываться. | M1 |
| Лидерство | Транслирует ценности. Формирует систему ценностей. | M1 |
| Лидерство | Распределяет задачи, учитывая потенциал сотрудников, их знания, опыт, сильные и слабые стороны. | M1 |
| Лидерство | Берет на себя ответственность за результат команды. | M1 |
| Лидерство | Отвечает за создание среды в команде, которая способствует высокой вовлеченности. Делает так, чтобы люди хотели работать в его команде. | M2 |
| Управление людьми | Является руководителем сотрудников своей команды. | M1 |

| Управление людьми | Регулярно доносит качественную обратную связь на личных встречах с сотрудниками. Применяет активное слушание. | M1 |
| Управление людьми | Мотивирует сотрудников команды. Использует подходящие инструменты мотивации для её повышения/поддержания на высоком уровне. | M1 |
| Управление людьми | Определяет сильные стороны и области развития членов команды. Помогает сотруднику определить возможные точки карьерного роста и достичь следующего желаемого грейда или должности (наставничеством, личным примером, обратной связью и пр.). | M1 |
| Управление людьми | Обсуждает с инженерами своей команды зарплату, вознаграждение и программы долгосрочной мотивации. Инициирует продвижение. Соблюдает принятый в компании процесс. | M1 |
| Управление людьми | Эффективно управляет конфликтами: выслушивает обе стороны и помогает разобраться. | M1 |
| Управление людьми | Ставит цели и задачи сотрудникам. | M1 |
| Управление людьми | Распределяет цели и задачи, учитывая их потенциал, знания, опыт, сильные и слабые стороны. | M2 |
| Управление командой | Помогает устранять возникающие внутри команды препятствия (во взаимодействии, процессах разработки и др.). | M1 |
| Управление командой | Инициирует найм и участвует в нём. Составляет профиль кандидата (описание для рекрутера) при необходимости с помощью techlead'а. Соблюдает существующий процесс. | M1 |
| Управление командой | Декомпозирует бизнес-требования на задачи для соответствующих платформ. Оценка задачи (анализ, описание, разработка, тестирование) совместно с членами своей команды. | M1 |
| Управление командой | Осуществляет промежуточный и итоговый контроль, корректирует процесс работы, её результат и даёт обратную связь команде. | M1 |
| Управление командой | Может проводить технические интервью в той области, где он эксперт, и менеджерское интервью, на котором проверяет soft skills кандидата. | M1 |
| Управление командой | Организует процесс онбординга для новичков. Ставит им цели на испытательный срок. | M1 |
| Управление командой | Инициирует процесс увольнения члена команды. Соблюдает принятый в компании процесс. | M1 |
| Управление командой | Своевременно каскадирует информацию, важную для команды. Например, рассказывает об основных изменениях в компании или изменениях в смежных юнитах/кластерах. | M2 |
| Управление командой | Обеспечивает команду необходимыми ресурсами для достижения цели. | M2 |
| Управление командой | Делится собственным опытом и управляет обменом знаниями внутри команды. | M2 |
| Управление командой | Имеет видение целевого состояния команды, на основе целей создания команды и целей департамента. Формирует её дизайн (состав, нужные компетенции и квалификация, отсутствие bus-фактора). Формирует план достижения целевого состояния. | M2 |
| Управление командой | Делает так, чтобы команда выполняла большую часть задач самостоятельно, без его непосредственного участия. Делегирует ответственность за принятие решений, но продолжает контролировать выполнение. | M2 |
| Управление командой | При поиске и решении проблем выходит за рамки команды, ищет системные проблемы на уровне выше. | M3 |
| Управление командой | Развивает не только людей, но и команду. Выращивает «зрелые» команды. | M3 |
| Целеполагание, планирование и реализация | Активно участвует и несёт ответственность за формирование квартальных планов на базе поставленных целей. | M1 |
| Целеполагание, планирование и реализация | Обеспечивает прозрачность целей, задач и планов внутри своей команды. | M1 |
| Целеполагание, планирование и реализация | Формирует и приоритизирует бэклог техдолга команды. На его основе составляет дорожную карту (roadmap). | M1 |
| Целеполагание, планирование и реализация | Основной стейкхолдер — product owner (второстепенно или где нет product owner — бизнес-заказчик). TeamLead своевременно транслирует важную для них информацию, управляет их ожиданиями. | M1 |
| Целеполагание, планирование и реализация | Выступает ответственным за достижение квартальных целей и планов команды. | M1 |
| Целеполагание, планирование и реализация | Выступает ответственным за постановку технических квартальных целей и планов команды. | M1 |
| Целеполагание, планирование и реализация | Обеспечивает горизонт планирования загрузки команды — от квартала до полугода. | M2 |
| Целеполагание, планирование и реализация | Несёт ответственность не только за команду и процессы, но и за результаты её работы и достижение поставленных целей. | M2 |
| Целеполагание, планирование и реализация | Для достижения целей меняет приоритеты при необходимости, предотвращает риски, корректирует действия команды для предотвращения нежелательных последствий. | M2 |
| Целеполагание, планирование и реализация | Отвечает за архитектурные изменения в сервисах/компонентах команды. Соблюдает принятый в компании процесс. | M2 |
| Целеполагание, планирование и реализация | Формирует миссию существования своей команды. Доносит эту миссию до всех. | M3 |
| Управление процессами | Работает в рамках существующих в компании инженерных процессов. Поддерживает их сам или делегирует команде процессы: Процесс разработки и тестирования; Управление инцидентами; Управление ошибками (багами); Управление техническим бэклогом; Управление документацией и знаниями; Управление релизным циклом. Соблюдает принятый в компании процесс. | M1 |
| Управление процессами | Проводит командные практики: дейли, планирование спринтов, ретро с командой по итогам инцидентов и спринтов, участвует в продуктовых QBR и груминге. | M1 |
| Управление процессами | Контролирует документацию по продукту в соответствии с текущим развитием продукта. Соблюдает DoR, DoD и DoS. | M1 |
| Управление процессами | Отвечает за предсказуемость и скорость процесса разработки и автоматизирует его. | M2 |
| Управление процессами | Устанавливает метрики эффективности процессов и следит за ними. | M3 |
| Управление процессами | Постоянно улучшает инженерные практики (процессы) в своей команде. Например: code review, архитектурные сессии, парное программирование, совместное программирование, разработка через тестирование (TDD). | M3 |
| Внедрение изменений | Может оставаться продуктивным, работая в состоянии неопределенности. | M1 |
| Внедрение изменений | При получении новой информации вносит изменения в ранее намеченный план, изменяет ранее принятые решения. | M1 |
| Внедрение изменений | Менеджер понимает важность изменений, быстро адаптируется к ним и умеет увидеть в них пользу для бизнеса. | M2 |
| Внедрение изменений | Внедряет изменения в команду и управляет ими, помогает своим сотрудникам адаптироваться. | M2 |
| Внедрение изменений | Анализирует информацию, предлагает новые идеи и решения для достижения цели, в том числе в ситуации неопределенности. | M2 |
| Техническая экспертиза | Наследует требования от Старший инженер из общего списка «Грейды линейных инженеров». Может тратить до 25% своего времени на работу руками. | M1 |
| Техническая экспертиза | Отвечает за техническое качество продуктов и сервисов, которые разрабатывает его команда. | M1 |
| Техническая экспертиза | Ведёт бэклог техдолга и дорожную карту продукта (roadmap). | M1 |
| Техническая экспертиза | Анализирует технические решения, предлагаемые командой. Привлекает techlead. Даёт конструктивную обратную связь по ним и принимает финальное решение. | M1 |
| Техническая экспертиза | Отвечает за стабильную работу системы/компонента своей команды. | M1 |
| Техническая экспертиза | Отвечает за то, чтобы система/компонент своей команды достаточно покрыта мониторингом и алертами. | M1 |
| Техническая экспертиза | Отвечает за то, что в команде есть дежурные, которые оперативно реагируют на инциденты. | M1 |
| Техническая экспертиза | Гарантирует соблюдение архитектурных паттернов на уровне приложения и интеграций самостоятельно либо совместно с techlead'ом и technical cluster lead'ом. | M1 |
| Техническая экспертиза | Развивает свои технические знания и навыки в соответствии со стеком технологий, которыми владеет его команда. При необходимости может самостоятельно решать задачи из бэклога команды в рамках своего технического бэкграунда (стека). | M2 |
| Бизнес-экспертиза | Вместе с product owner формирует бэклог продукта и дорожную карту (roadmap). | M1 |
| Бизнес-экспертиза | Руководствуется знаниями сценариев использования своего продукта при принятии решений. | M2 |
| Бизнес-экспертиза | Вместе с product owner формирует и отслеживает метрики здоровья продукта. | M2 |
| Бизнес-экспертиза | При проработке задач предлагает альтернативные способы проверки гипотез и технических решений, позволяющих получить данные быстрее или с меньшими затратами. | M2 |
| Бизнес-экспертиза | Исследует проблемы пользователей и помогает заказчику найти бизнес-решение, например, участвует в Discovery-процессе. | M3 |
| Бизнес-экспертиза | Делает базовые выводы по результатам проведённого исследования, например, по A/B-тестам: определение статистически значимого результата или нет; понимание того, что тест показывает плохие результаты на метриках и его нужно остановить. «Подсвечивает» найденные проблемы или успехи. | M3 |
